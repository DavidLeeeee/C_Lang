# 01_array_pointer 학습 포인트

대상 파일: `01_array_pointer.c`

## 이 파일이 시사하는 핵심

이 예제의 핵심 메시지는 하나다.

**배열(array)과 포인터(pointer)는 매우 비슷하게 보이지만, 같은 것은 아니다.**

C를 처음 공부할 때 가장 많이 헷갈리는 지점이 바로 이것이다. 이 파일은 그 차이를 직접 출력으로 확인하게 만든다.

---

## 이 예제가 보여 주는 3가지 핵심 사실

### 1. `arr`는 배열이고, `ptr`는 포인터 변수다

`main` 안에서:

```c
int arr[5] = {10, 20, 30, 40, 50};
int *ptr = arr;
```

여기서 `arr`는 **정말로 5개의 `int`를 담는 배열 객체**다.  
반면 `ptr`는 **어떤 `int`를 가리키는 포인터 변수**다.

즉:

- `arr`는 메모리 공간 자체를 가진 배열
- `ptr`는 주소값 하나를 저장하는 변수

겉으로는 둘 다 `arr[2]`, `ptr[2]`처럼 비슷하게 사용할 수 있지만, 정체는 다르다.

---

### 2. 배열은 많은 상황에서 포인터처럼 변한다

이 코드에서:

```c
int *ptr = arr;
```

배열 `arr`는 표현식에서 자주 첫 번째 원소의 주소로 변환된다. 이를 보통 **array-to-pointer decay**라고 부른다.

그래서:

- `arr`
- `ptr`
- `&arr[0]`

이 셋을 출력하면 비슷한 주소처럼 보인다.

하지만 이건 "같다"는 뜻이 아니라, **배열이 특정 문맥에서 포인터처럼 변환되어 사용된다**는 뜻이다.

---

### 3. 함수 인자로 넘어가면 배열은 사실상 포인터로 취급된다

이 함수 선언을 보자.

```c
static void inspect_array_parameter(int arr_param[], size_t count)
```

겉보기에는 배열 파라미터처럼 보이지만, 실제로는 거의 아래와 동일하게 동작한다.

```c
static void inspect_array_parameter(int *arr_param, size_t count)
```

그래서 함수 안에서:

```c
sizeof(arr_param)
```

를 하면 배열 전체 크기가 아니라 **포인터 크기**가 나온다.

네가 실제로 본 경고도 바로 이 지점을 설명한다.

- 함수 파라미터의 `arr_param[]`는 진짜 배열이 아님
- `sizeof(arr_param)`는 `int *` 크기임
- 그래서 GCC가 경고를 띄움

이 경고는 에러가 아니라, **이 예제의 핵심을 알려 주는 학습용 신호**다.

---

## 왜 이게 중요한가

이 개념을 모르면 C에서 아래 실수를 아주 자주 한다.

- 함수 안에서 `sizeof(param)`로 배열 길이를 구하려고 함
- 문자열 버퍼 길이를 잘못 계산함
- 배열과 포인터를 완전히 같은 것으로 착각함
- 함수 인자로 넘긴 뒤 크기 정보가 사라지는 이유를 이해하지 못함

즉 이 예제는 단순 문법 예제가 아니라, 앞으로 C에서 계속 등장하는 버그의 출발점을 보여 준다.

## 이 파일을 통해 기억해야 할 문장

### 꼭 기억할 핵심 문장 1

**배열은 포인터가 아니다.**

### 꼭 기억할 핵심 문장 2

**하지만 배열은 많은 표현식에서 첫 원소를 가리키는 포인터처럼 decay된다.**

### 꼭 기억할 핵심 문장 3

**함수 파라미터의 `T arr[]`는 사실상 `T *arr`처럼 동작한다.**

## 코드별 해석 포인트

### `sizeof(arr)`

- 배열 전체 바이트 수
- 예: `5 * sizeof(int)`

### `sizeof(ptr)`

- 포인터 변수 자체의 바이트 수
- 보통 64비트 환경에서는 8바이트

### `arr + 1`, `ptr + 1`

둘 다 다음 `int` 원소 위치로 이동한다.  
즉 포인터 연산은 바이트가 아니라 **원소 크기 기준**으로 움직인다.

### `*(arr + 2)`와 `ptr[2]`

두 표현은 같은 원소를 읽는다. 이 때문에 배열과 포인터가 더 헷갈리기 쉽다.  
하지만 "사용 방식이 비슷하다"와 "같은 존재다"는 다르다.

### `inspect_array_parameter(arr, ...)`

이 호출에서 함수는 배열 전체 객체를 그대로 보존해서 받는 것이 아니라,  
배열의 첫 원소 주소를 기준으로 접근하게 된다.

그래서 길이 정보는 자동으로 따라가지 않는다.  
이 때문에 `count`를 따로 넘기는 패턴이 C에서 매우 흔하다.
