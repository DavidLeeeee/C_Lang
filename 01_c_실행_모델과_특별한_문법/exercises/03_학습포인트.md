# 03_storage_duration 학습 포인트

대상 파일: `03_storage_duration.c`

## 이 파일이 시사하는 핵심

이 예제의 핵심 메시지는 아래 한 줄이다.

**`static`의 핵심은 "어디서 보이느냐(scope)"보다 "얼마나 오래 사느냐(storage duration)"에 있다.**

즉 함수 안에 선언했더라도 `static`이면 호출이 끝나도 값이 사라지지 않는다.  
반대로 일반 지역 변수(`int local_counter`)는 함수 호출이 끝나면 수명이 끝난다.

---

## 이 예제가 보여 주는 4가지 핵심 사실

### 1. 일반 지역 변수는 호출마다 새로 만들어진다

```c
int local_counter = 0;
local_counter++;
```

`local_counter`는 자동 저장 기간(automatic storage duration) 변수다.

- `tick()` 진입 시 생성
- `tick()` 종료 시 소멸
- 다음 호출에서 다시 0으로 시작

그래서 출력이 항상 `1`이다.

---

### 2. 함수 내부 `static` 변수는 호출 사이에 값을 유지한다

```c
static int function_static_counter = 0;
function_static_counter++;
```

이 변수는 함수 안에 있지만 정적 저장 기간(static storage duration)을 가진다.

- 프로그램 시작 시점에 한 번 초기화
- 프로그램 종료까지 유지
- `tick()`을 여러 번 호출해도 같은 메모리 객체를 계속 사용

그래서 출력이 `1`, `2`, `3`으로 누적된다.

---

### 3. `global`과 `file static`의 lifetime은 같다

```c
int global_counter = 100;
static int file_static_counter = 1000;
```

둘 다 정적 저장 기간이라 프로그램 전체 lifetime 동안 살아 있다.  
차이는 lifetime이 아니라 **linkage(연결성)**다.

- `global_counter`: external linkage (다른 번역 단위에서 `extern`으로 참조 가능)
- `file_static_counter`: internal linkage (현재 `.c` 파일 내부에서만 보임)

---

### 4. `static`은 "메모리 위치"를 보장하는 키워드가 아니다

학습 단계에서 종종 "`static`이면 데이터 섹션", "지역 변수면 스택"처럼 단순화해서 외우기 쉽다.  
하지만 C 표준이 보장하는 것은 물리적 메모리 구역 이름이 아니라 **저장 기간과 linkage 규칙**이다.

즉 중요한 것은:

- 언제 생성/소멸되는가
- 어디서 이름으로 접근 가능한가

---

## 질문에 대한 직접 답변

### 1. 왜 `local_counter`는 항상 1인가?

호출마다 새 객체가 생성되고 `0 -> 1`만 수행한 뒤 소멸하기 때문이다.

### 2. 왜 `function_static_counter`는 호출 사이에 유지되는가?

함수 스코프 안에 있지만 정적 저장 기간 변수라 프로그램 전체 동안 살아 있기 때문이다.

### 3. file static과 global의 차이는 lifetime이 아니라 무엇인가?

둘 다 lifetime은 동일(프로그램 전체)이고, 차이는 linkage다.

- file static: 파일 내부 전용
- global: 파일 바깥에서도 참조 가능

---

## 왜 이게 중요한가

이 개념을 정확히 구분하지 못하면 아래 실수가 반복된다.

- 함수 내부 `static`을 지역 변수처럼 생각해 값 초기화를 잘못 가정
- 다중 호출 함수에서 상태가 누적되는 버그를 놓침
- 파일 경계에서 심볼 충돌/노출 문제를 제어하지 못함

즉 저장 기간과 linkage 구분은 C 실행 모델 이해의 핵심이다.

## 자주 하는 실수

### 오해 1

"함수 안에 있으니 `static`도 함수가 끝나면 사라진다"

아니다. 함수 안에 있어도 `static`이면 프로그램 종료까지 유지된다.

### 오해 2

"`global`은 오래 살고 `file static`은 짧게 산다"

아니다. 둘 다 정적 저장 기간이다. 차이는 접근 가능 범위(linkage)다.

### 오해 3

"지역 변수 `int`는 런타임 전체에서 유지된다"

아니다. 자동 저장 기간 변수는 해당 블록(보통 함수 호출) 실행 동안만 유지된다.

`tick()`에 아래 두 변수를 추가해 출력해 보자.

```c
int auto_counter = 0;
static int static_counter = 0;
```

둘 다 `++` 후 3회 호출 결과를 비교하면, 자동 저장 기간과 정적 저장 기간 차이가 즉시 눈에 보인다.

## 한 줄 요약

**함수 안 `static`은 "보이는 범위는 지역, 수명은 전역"이고, 일반 지역 변수는 "보이는 범위도 지역, 수명도 호출 동안"이다.**
